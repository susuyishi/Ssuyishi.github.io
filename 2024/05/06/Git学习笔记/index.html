Git学习笔记

Created2024-05-06|Updated2024-05-06
|Post Views:

做一个简单的记录
从菜鸟教程-Git上进行学习

1. Git的安装与配置

1.1 安装Git

Windows系统安装Git，可以从Git官网下载安装程序，安装时可以选择默认安装路径，然后在开始菜单里找到Git Bash，打开命令行工具，输入git命令，如果出现版本信息，则说明Git安装成功。

1.2 配置Git

Git 提供了一个叫做 git config 的工具，专门用来配置或读取相应的工作环境变量。

这些环境变量，决定了 Git 在各个环节的具体工作方式和行为。这些变量可以存放在以下三个不同的地方：

/etc/gitconfig 文件：系统中对所有用户都普遍适用的配置。若使用 git config 时用 –system 选项，读写的就是这个文件。
~/.gitconfig 文件：用户目录下的配置文件只适用于该用户。若使用 git config 时用 –global 选项，读写的就是这个文件。
当前项目的 Git 目录中的配置文件（也就是工作目录中的 .git/config 文件）：这里的配置仅仅针对当前项目有效。每一个级别的配置都会覆盖上层的相同配置，所以 .git/config 里的配置会覆盖 /etc/gitconfig 中的同名变量。
Git的配置包括三个方面：

  • 用户信息：用来标识提交者的相关信息，包括姓名、邮箱、用户名等。 如果用了 –global 选项，那么更改的配置文件就是位于你用户主目录下的那个，以后你所有的项目都会默认使用这里配置的用户信息。

如果要在某个特定的项目中使用其他名字或者电邮，只要去掉 –global 选项重新配置即可，新的设定保存在当前项目的 .git/config 文件里。

  • 文本编辑器：用来配置Git命令行工具使用的文本编辑器，比如vim、nano等。
  • 差异分析工具：用来配置Git命令行工具使用的差异分析工具，比如vimdiff、tkdiff等。
  • 查看配置信息：可以用 git config --list 命令查看所有配置信息。

2. Git工作流程

2.1 克隆仓库

克隆仓库是指将一个远程仓库复制到本地，这样就可以在本地进行开发工作。

克隆仓库的命令是 git clone [url]，其中 [url] 是远程仓库的地址。

2.2 基本工作流程

Git的工作流程一般分为以下几个步骤：

  • 工作区：就是你在电脑里能看到的目录，通过编辑器或者 IDE 存放代码。
  • 暂存区（Index）：暂存区是一个文件，保存了下次将提交的文件列表信息，一般在 Git 仓库目录中。
  • 本地仓库：就是 Git 仓库，保存了版本历史，包括所有的提交记录。
  • 远程仓库：托管在远程服务器上的仓库，可以让多个开发者协同开发。

2.3 添加文件

要向暂存区添加一个文件，有以下三种方式：

  • git add [file]：添加指定文件到暂存区。
  • git add .：添加当前目录的所有文件到暂存区。
  • git add -u：添加当前目录下被修改的文件到暂存区。

2.4 提交更改

提交更改有以下三种方式：

  • git commit -m "commit message"：提交暂存区到本地仓库。
  • git commit -a -m "commit message"：跳过暂存区，直接将所有已跟踪过的文件暂存起来一并提交。
  • git commit --amend -m "commit message"：修改最后一次提交。
