Git学习笔记

Created2024-05-06|Updated2024-05-07
|Post Views:

做一个简单的记录
从菜鸟教程-Git上进行学习

1. Git的安装与配置

1.1 安装Git

Windows系统安装Git，可以从Git官网下载安装程序，安装时可以选择默认安装路径，然后在开始菜单里找到Git Bash，打开命令行工具，输入git命令，如果出现版本信息，则说明Git安装成功。

1.2 配置Git

Git 提供了一个叫做 git config 的工具，专门用来配置或读取相应的工作环境变量。

这些环境变量，决定了 Git 在各个环节的具体工作方式和行为。这些变量可以存放在以下三个不同的地方：

/etc/gitconfig 文件：系统中对所有用户都普遍适用的配置。若使用 git config 时用 –system 选项，读写的就是这个文件。
~/.gitconfig 文件：用户目录下的配置文件只适用于该用户。若使用 git config 时用 –global 选项，读写的就是这个文件。
当前项目的 Git 目录中的配置文件（也就是工作目录中的 .git/config 文件）：这里的配置仅仅针对当前项目有效。每一个级别的配置都会覆盖上层的相同配置，所以 .git/config 里的配置会覆盖 /etc/gitconfig 中的同名变量。
Git的配置包括三个方面：

  • 用户信息：用来标识提交者的相关信息，包括姓名、邮箱、用户名等。 如果用了 –global 选项，那么更改的配置文件就是位于你用户主目录下的那个，以后你所有的项目都会默认使用这里配置的用户信息。

如果要在某个特定的项目中使用其他名字或者电邮，只要去掉 –global 选项重新配置即可，新的设定保存在当前项目的 .git/config 文件里。

  • 文本编辑器：用来配置Git命令行工具使用的文本编辑器，比如vim、nano等。
  • 差异分析工具：用来配置Git命令行工具使用的差异分析工具，比如vimdiff、tkdiff等。
  • 查看配置信息：可以用 git config --list 命令查看所有配置信息。

2. Git工作流程

2.1 基本工作流程

Git的工作流程一般分为以下几个步骤：

  • 克隆仓库：将远程仓库克隆到本地。
  • 创建分支：在本地仓库创建分支，以便进行开发。
  • 切换分支：在不同分支之间进行切换。
  • 开发工作：在当前分支上进行开发工作。
  • 添加更改：将开发工作成果添加到暂存区。
  • 提交更改：将暂存区的更改提交到本地仓库。
  • 推送分支：将本地分支的更改推送到远程仓库。
  • 合并分支：将其他分支的更改合并到当前分支。
  • 删除分支：删除已经不需要的分支。

2.2 基本概念

  • 工作区：就是在电脑里能看到的目录，通过编辑器或者 IDE 存放代码。工作区中的文件可能处于不同的状态：已修改但未暂存、已暂存但未提交，或已提交到版本库。
  • 暂存区（Index）：暂存区是一个文件，保存了下次将提交的文件列表信息，一般在 Git 仓库目录中。
  • 本地仓库：就是 Git 仓库，保存了版本历史，包括所有的提交记录。
  • 远程仓库：托管在远程服务器上的仓库，可以让多个开发者协同开发。

3. IDE与Git的集成(git命令敲到一半用这个来了 )

集成Git的IDE有很多，如：Eclipse、NetBeans、IntelliJ IDEA、Visual Studio Code等。

  • 集成Git的步骤：
  • 安装Git：安装Git，并配置好环境变量。
  • 配置IDE：配置IDE，指定Git的路径。
  • 使用IDE：使用IDE，编写代码，并提交到Git仓库。

4.Git命令

这里做一些常用的Git命令的记录，更多命令请参考菜鸟教程-Git

3.1 创建仓库

  • git init：初始化一个新的 Git 仓库。
    git init [project-name]：在指定目录下初始化一个新的 Git 仓库。
  • git clone [url]：克隆一个远程 Git 仓库。
    git clone [url] [project-name]：克隆一个远程 Git 仓库到指定目录。

3.2 查看状态

以下命令也可以通过-s选项查看简略的状态信息：

  • git status：查看仓库当前的状态。
  • git diff：查看工作区和暂存区的差异。
  • git diff --cached：查看暂存区和版本库的差异。
  • git log：查看提交历史。
  • git blame [file]: 查看指定文件历史记录。

3.3 添加文件

要向暂存区添加一个文件，有以下三种方式：

  • git add [file]：添加指定文件到暂存区。
  • git add .：添加当前目录的所有文件到暂存区。
  • git add -u：添加当前目录下被修改的文件到暂存区。

3.4 提交更改

提交更改有以下三种方式：

  • git commit -m "commit message"：提交暂存区到本地仓库。
  • git commit -a -m "commit message"：跳过暂存区，直接将所有已跟踪过的文件暂存起来一并提交。
  • git commit --amend -m "commit message"：修改最后一次提交。

3.5 分支管理(必杀技特性)

使用分支意味着可以从开发主线上分离开来，然后在不影响主线的同时继续工作。

  • git branch：查看本地分支。
  • git branch [branch-name]：创建分支。
  • git checkout [branch-name]：切换分支。
  • git checkout -b [branch-name]：创建分支并切换到该分支。
  • git merge [branch-name]：合并指定分支到当前分支。
  • git branch -d [branch-name]：删除分支。
  • git branch -D [branch-name]：强制删除分支。

3.6 标签管理

发布一个版本时，我们通常先在版本库中打一个标签（tag），这样就唯一确定了打标签时刻的版本。将来无论什么时候，取某个标签的版本，就是把那个打标签的时刻的历史版本取出来。

所以，标签也是版本库的一个快照。

  • git tag：查看所有标签。
  • git tag [tag-name]：创建轻量标签。
  • git tag -a[tag-name]：创建带注解的标签。
  • git tag -d [tag-name]：删除标签。
  • git show [tag-name]：显示指定标签信息。
  • git push [remote-name] [tag-name]：推送标签到远程仓库。
  • git push --tags：推送所有标签到远程仓库。
  • git push [remote-name] --delete [tag-name]：删除远程标签。
  • git push origin --tags：推送所有标签到远程仓库。
  • git fetch --tags：从远程仓库拉取所有标签。

3.7 远程仓库管理

  • git remote：查看远程仓库。
  • git remote add [remote-name] [url]：添加远程仓库。
  • git remote rm [remote-name]：删除远程仓库。
  • git remote show [remote-name]：显示远程仓库信息。
  • git fetch [remote-name]：从远程仓库拉取数据。
  • git pull [remote-name] [branch-name]：从远程仓库拉取分支。
  • git push [remote-name] [branch-name]：推送本地分支到远程仓库。
  • git push [remote-name] --tags：推送所有标签到远程仓库。
  • git push [remote-name] --delete [branch-name]：删除远程分支。
  • git push [remote-name] --set-upstream [branch-name]：设置远程分支。
  • git push [remote-name] --force：强制推送当前分支到远程仓库。
  • git clone --depth=1 [url]：克隆指定分支的历史记录。
  • git gc：压缩仓库。

3.8 其他命令

  • git rm [file]：删除文件。
  • git reset [file]：取消暂存文件。
  • git stash：保存当前工作现场。
  • git stash pop：恢复工作现场。
  • git config：配置 Git。
  • git help：显示 Git 命令的帮助信息。
