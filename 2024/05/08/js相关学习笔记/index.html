前言

先写着再说

HTML DOM

HTML DOM 定义了所有 HTML 元素的对象和属性，以及访问它们的方法。

节点

DOM的所有事物都是节点包括

  • 元素节点
  • 属性节点
  • 文本节点
  • 注释节点
  • 文档节点
    节点彼此拥有层级关系，常用父节点、子节点和同胞节点。
    HTML DOM树

方法

在节点上执行的动作

方法 描述
getElementById() 返回带有指定 ID 的元素。
getElementsByTagName() 返回包含带有指定标签名称的所有元素的节点列表（集合/节点数组）。
getElementsByClassName() 返回包含带有指定类名的所有元素的节点列表。
appendChild() 把新的子节点添加到指定节点。
removeChild() 删除子节点。
replaceChild() 替换子节点。
insertBefore() 在指定的子节点前面插入新的子节点。
createAttribute() 创建属性节点。
createElement() 创建元素节点。
createTextNode() 创建文本节点。
getAttribute() 返回指定的属性值。
setAttribute() 把指定属性设置或修改为指定的值。

访问

  1. getElementById()
  2. getElementsByTagName()
  3. getElementsByClassName()

元素

添加、删除和替换 HTML 元素。

事件

DOM事件是用户与网页的交互行为，如点击onclick、输入oninput、移动onmousemove、加载onload等。
事件需要分配到元素上，当事件发生时，执行相应的函数。

属性

节点中获取或设置的值
常用属性：

  • innerHTML - 节点（元素）的文本值
  • parentNode - 节点（元素）的父节点
  • childNodes - 节点（元素）的子节点
  • attributes - 节点（元素）的属性节点
  • nodeName - 节点的名称
  • nodeType - 节点的类型
  • nodeValue - 节点的值
  • firstChild - 节点的第一个子节点
  • lastChild - 节点的最后一个子节点
  • nextSibling - 节点的下一个同胞节点
  • previousSibling - 节点的上一个同胞节点

导航

DOM提供一些方法来导航文档树，如getElementsByTagName() 方法返回节点列表。
节点列表是一个节点数组。

Node.js

Node.js 是一个基于 Chrome V8 引擎的 JavaScript 运行环境。
Node.js 使用事件驱动、非阻塞 I/O 模型，使其轻量又高效，非常适合运行实时的 web 应用。
Node.js 包管理器 npm 简化了模块依赖管理。

安装

Node.js 下载地址：https://nodejs.org/en/download/
安装完成后，打开命令行，输入node -v命令查看版本号，确认安装成功。

基本语法

Node.js 代码以模块化方式编写，每一个文件就是一个模块。
模块通过require()函数引入其他模块。

模块

Node.js 内置了很多模块，可以直接使用。

  • 文件系统模块：fs
  • 路径模块：path
  • 网络模块：net
  • HTTP 模块：http
  • 数据库模块：mysql、mongodb
  • 压缩模块：zlib
  • 加密模块：crypto
  • 单元测试模块：assert
  • 控制台模块：console
  • 进程模块：process
  • 全局对象：global
  • 定时器模块：setTimeout、setInterval
  • 事件模块：EventEmitter
  • 系统模块：os
  • 路径模块：path
  • 全局对象：global
  • 网络模块：net

异步 I/O

Node.js 采用事件驱动、非阻塞 I/O 模型，所有 I/O 操作都不会阻塞线程，可以实现高并发处理。
异步 I/O 编程模型的特点：

  • 事件循环：Node.js 使用事件循环模型，所有 I/O 操作都在事件循环中完成，不阻塞线程。
  • 回调函数：Node.js 采用回调函数，将异步操作的结果或错误通过回调函数返回。
  • 错误优先回调：Node.js 采用错误优先回调，即发生错误时，将错误对象传入回调函数。

这里谈谈阻塞和非阻塞 I/O与同步和异步的区别，我举个简单的例子来说明：

我要看足球比赛，但是妈妈叫我烧水，电视机在客厅，烧水要在厨房。家里有2个水壶，一个是普通的水壶，另一个是水开了会叫的那种水壶。我可以：

  • 用普通的水壶烧，人在边上看着，水开了再去看球。（同步，阻塞）这个是常规做法，但是我看球不爽了。
  • 用普通水壶烧，人去看球，隔几分钟去厨房看看。（同步，非阻塞）这个又大问题，万一在我离开的几分钟水开了，我就麻烦了。
  • 用会叫的水壶，人在边上看着。（异步，阻塞）这个没有问题，但是我太傻了。
  • 用会叫的水壶，人去看球，听见水壶叫了再去看。（异步，非阻塞）这个应该是最好的。
    等着看球的我：阻塞
    看着电视的我：非阻塞
    普通水壶：同步
    会叫的水壶：异步

所以，异步往往配合非阻塞，才能发挥出威力。
